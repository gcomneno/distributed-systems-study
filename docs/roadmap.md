# Study Roadmap

The roadmap prioritizes concepts useful to backend engineering,
software architecture, and System Design interviews.

## Block A — Foundations

1. What makes a system distributed
2. Remote communication, RPC, messages, and retries
3. System models, channels, synchrony, and assumptions
4. Atomicity, fairness, safety, and liveness
5. Cumulative review and timeout ambiguity laboratory

## Block B — Time, order, and global state

6. Physical time, logical clocks, and vector clocks
7. Clock drift, timeouts, and failure suspicion
8. Consistent cuts and distributed snapshots
9. Reliable, causal, and total-order multicast
10. Cumulative review and interview session

## Block C — Coordination and failures

11. Distributed mutual exclusion
12. Leader election
13. Fault models and fault-tolerant systems
14. Consensus and impossibility results
15. Paxos and failure detectors

## Block D — Distributed data

16. Distributed transactions and atomic commit
17. Replication and consistency models
18. Quorums, CAP, and Dynamo-style systems
19. Cumulative System Design exercise

## Block E — Final project

20. Requirements, assumptions, and failure budget
21. Partitioning, replication, consistency, retry, and idempotency
22. Recovery, observability, capacity, and final interview

## Optional modules

- Distributed termination and deadlock detection
- Routing and distributed graph algorithms
- Self-stabilizing systems
- Peer-to-peer networks and distributed hash tables
- Security in distributed systems
